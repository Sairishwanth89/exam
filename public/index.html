About ProctorGuard AI

ProctorGuard AI is a secure online exam monitoring platform built to help institutions run remote assessments with more confidence. It combines browser-based exam access, AI-assisted camera checks, and centralized review tools so the exam process feels professional rather than improvised.

Official exam entry point

The homepage introduces the platform and gives students and admins a clean path into their account flow.

Secure and structured

The system uses login, role handling, and AI proctoring to keep the experience organized and auditable.

Professional presentation

The layout is designed to resemble an official software product site with clear trust signals and concise messaging.

Why teams use it

Remote exams fail when the platform is too heavy, too noisy, or too rigid. This project is built to stay light in the browser, keep the exam flow simple, and surface only meaningful alerts such as face absence, multiple faces, head movement, phone usage, earbuds, and headsets.

Solves common exam risks

Detects the behaviors that usually break trust in remote tests.

  • Face not visible or student leaves the frame
  • Multiple faces or identity mismatch
  • Mobile phones, earbuds, and headphones in view
  • Suspicious eye movement and head rotation patterns

Designed for practical usage

Fits into a standard web stack and works with the existing exam flow.

  • Express backend with exam session handling
  • Python AI worker for camera analysis
  • MongoDB-backed persistence for exam and proctoring data
  • Browser-first workflow, no native app install needed

How it works

The flow is intentionally simple: students log in, verify their face, start the exam, and the AI service evaluates each frame in the background.

Step 01 Login and enroll

Student credentials are loaded, then the camera is used to confirm identity before the exam starts.

Step 02 Start exam session

The exam page opens with a visible timer, live webcam monitor, and a clear integrity widget.

Step 03 Analyze in real time

The Python service checks face presence, gaze, head pose, and nearby devices from each frame.

Step 04 Record and review

Suspicious actions become structured events that can be reviewed after the submission.
